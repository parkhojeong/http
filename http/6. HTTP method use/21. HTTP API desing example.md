[MEMBER MANAGER SYSTEM - Collection]
server has to know resource location

API design
- list: /members -> GET
- create: /members -> POST
- query member: /members/{id} -> GET
- update member: /members/{id} -> PATCH, PUT, POST
 - PATCH: partial update
 - PUT: full update(ex. update board article)
 - POST: etc
- delete member: /members/{id} -> DELETE

/members - collection(server directory)
/members/{id} - member

create based on POST 
- server decides resource to create
- return resource location header
 - HTTP/1.1 201 Created
 - Location: /members/100 
- [Collection]
 - server manages resource directory
 - server knows and decides resource location
 - /members is collection

[FILE MANAGER SYSTEM - Store]
client has to know resource location

API design
- list: /files -> GET
- query file: /files/{id} -> GET
- create file: /files/{id} -> PUT
 - client decides file name
 - client knows file name
- delete file: /files/{id} -> DELETE
- bulk upload: /files -> POST

feature
- client has to know resource location
 - /files/{filename} -> PUT
 - PUT /files/star.jpg
- client decides uri 
- [Store]
 - client manages resource store
 - client knows and decides resource location 
 - /files is store

[HTML FORM - MEMBER MANAGER SYSTEM]
- support only GET, POST
- control URI: resource location with verb
 - ex. /members/new, /members/{id}/edit

API design
- list         : /members -> GET
- create form  : /members/new -> GET
- create member: /members/new, /members -> POST
 - /members -> collection method 
 - /members/new -> control uri
- query member : /members/{id} -> GET
- update form  : /members/{id}/edit -> GET
- update member: /members/{id}/edit, /members/{id} -> POST
- delete member: /members/{id} -> POST
 - control uri 

URI design rule
https://restfulapi.net/resource-naming
- 1. document
 - single resource(one file, object instance, database row)
 - ex. /members/100, files/star.jpg
- 2. collection
 - server manages resource directory
 - server knows and decides resource location
 - ex. /members  
- 3. store
 - client manages resource store
 - client knows and decides resource location 
 - ex. /files
- 4. control uri, controller
 - (document, collection, store) instead
 - use verb
 - ex. /members/{id}/new
