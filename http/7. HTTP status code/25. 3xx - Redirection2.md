[temporary redirection]
302, 307, 303

- 302(mostly used)
 - redirect with GET, no body(MAY)
- 307
 - redirect with same method, body(MUST!) 
- 303
 - redirect with GET


PRG(POST/Redirect/GET)
== not application PRG start ==
[request]
POST /order HTTP/1.1
Host: www.example.com

itemId=mouse&count=1
[response]
HTTP/1.1 200 OK

<html>order completed</html>

[request by browser refresh]
GET /order HTTP/1.1
Host: www.example.com

itemId=mouse&count=1
== not application PRG end ==

== application PRG start ==
[request]
POST /order HTTP/1.1
Host: www.example.com

itemId=mouse&count=1
[response]
HTTP/1.1 302 Found
Location: /order-result/19

[redirect]
GET /order-result/19 HTTP/1.1
Host: www.example.com


summary
- 302 FOUND: may change GET(mostly used)
- 307 TEMPORARY REDIRECT: must not change method(recommended)
- 303 SEE OTHER: change GET(recommended)

other
- 300 Multiple Choices: not used
- 304 Not Modified:
 - for cache
 - notify client that resource is not updated. client use cache in local storage.(redirect cache)
 - response body must be empty
 - for conditional GET, HEAD request
