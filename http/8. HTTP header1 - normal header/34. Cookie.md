Set-Cookie(Response)
- ex. set-cookie: sessionid=123456; expires=Thu, 18 Dec 2013 12:00:00 GMT; path=/; domain=.example.com; Secure
- usage
 - user login session management
 - advertising information tracking
- cookie always send to server
 - network traffic
 - use minimum information (session id, authentication token)
- caution
 - security issue

Cookie(Request)


[life cycle]
expires, max-age
- Set-Cookie: expires=Thu, 18 Dec 2013 12:00:00 GMT
    - expired time = expires time
- Set-Cookie: max-age=3600
    - expired time = current time + 3600 seconds
- Session cookie
    - expired time = browser closed
- persistent cookie
    - expired time = expires time

[Domain]
ex) example.com
- specify domain: domain and sub domain
  - domain=example.com
    - example.com, www.example.com, ...
- omission: target domain only
    - example.com only

[Path]
ex) path=/home
- path and sub path
- generally path=/ (root)
- ex)
  - path=/home
  - /home -> possible
  - /home/level1 -> possible
  - /level2 -> impossible

[Security]
Secure, HttpOnly, SameSite
- Secure: https only
- HttpOnly
  - XSS blocking
  - javascript can't access cookie(document.cookie)
  - http request only
- SameSite
  - XSRF blocking
  - send only (request domain=cookie domain)
  - check browser version(recent browser support)
