Method type
- GET: query 
- POST: create, handling request data
- PUT: replace or create new resource
- PATCH: partial update
- DELETE: delete 
OTHER
- HEAD: GET response without body
- OPTIONS: OPTION request (check what method can be used)
- CONNECT: tunneling
- TRACE: debug

GET
- query resource
- send data by query(query string, query parameter)
- body possible, but not recommended

POST
- handle request data
 - create new resource -> response 201 with location header
 - handling request data -> consider control URI(/orders/{orderId}/start-delivery)
 - ambiguous case 
- send data by body
